# Grok Register 启动指南

本文说明如何在本机安装、配置并启动本项目（GUI / CLI）。  
主入口文件：`grok_register_ttk.py`。

> 本项目仅用于自动化流程研究、测试环境验证和个人学习。请遵守目标网站服务条款与当地法律法规。

---

## 1. 环境要求

| 项目 | 要求 |
| --- | --- |
| Python | **3.9+**（推荐 3.11） |
| 浏览器 | Google Chrome 或 Chromium |
| 网络 | 可访问注册页与所选邮箱 API（可配置代理） |
| GUI | 需要 Tkinter；没有时可只用 CLI |

---

## 2. 安装

### 2.1 进入项目目录

```text
D:\chenjing\AI_PROJECT\grok-register
```

或克隆后进入仓库根目录。

### 2.2 创建虚拟环境（首次）

```bash
python -m venv .venv
```

### 2.3 安装依赖

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 2.4 准备配置文件

```bash
# Windows CMD
copy config.example.json config.json

# PowerShell
Copy-Item config.example.json config.json

# macOS / Linux / Git Bash
cp config.example.json config.json
```

编辑 `config.json`，至少配置：

- `email_provider`：`duckmail` / `yyds` / `cloudflare` / `cloudmail`
- 对应邮箱服务的 API Key / JWT / 域名等
- 可选：`proxy`、`register_count`、CPA / grok2api 相关项

`config.json` 已在 `.gitignore` 中，不会被提交。

配置校验分两层：

1. **结构校验**：GUI 启动时执行，类型/枚举/范围检查，便于打开界面改配置。
2. **运行校验**：点击「开始注册」或 CLI 真正开跑前，检查当前启用功能所需字段。

## 3. 启动命令（按终端区分）

不同终端语法不同。**不要混用**：在 PowerShell 里不要用 `source` 和 `&&`（旧版 PowerShell 会报错）。

### 3.1 Windows PowerShell（推荐本机日常使用）

```powershell
cd D:\chenjing\AI_PROJECT\grok-register
.\.venv\Scripts\Activate.ps1
python grok_register_ttk.py
```

若提示无法运行脚本（执行策略限制）：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
python grok_register_ttk.py
```

**不激活虚拟环境**，直接调用 venv 里的 Python：

```powershell
.\.venv\Scripts\python.exe grok_register_ttk.py
```

### 3.2 Windows CMD

```bat
cd /d D:\chenjing\AI_PROJECT\grok-register
.venv\Scripts\activate.bat
python grok_register_ttk.py
```

### 3.3 Git Bash

```bash
cd /d/chenjing/AI_PROJECT/grok-register
source .venv/Scripts/activate
python grok_register_ttk.py
```

或一行：

```bash
source .venv/Scripts/activate && python grok_register_ttk.py
```

### 3.4 macOS / Linux

```bash
cd /path/to/grok-register
source .venv/bin/activate
python grok_register_ttk.py
```

## 4. 运行模式

### 4.1 GUI（默认）

```bash
python grok_register_ttk.py
```

- 启动后读取 `config.json` 并做结构校验。
- 在界面中确认配置，点击 **「开始注册」** 后做运行校验，再启动后台线程。
- 每个新批次开始前：成功 / 失败 / 待恢复 / 后处理警告 四项统计会清零。

### 4.2 CLI（无 Tk 界面）

以下三种写法等价：

```bash
python grok_register_ttk.py cli
python grok_register_ttk.py start
python grok_register_ttk.py --cli
```

流程：

1. 加载 `config.json` 并做运行校验  
2. 终端提示后输入 `start` 才开始  
3. `Ctrl+C` 可请求停止并清理  

> CLI 只是不启动 Tk GUI，注册过程仍会打开 Chromium / Chrome。

### 4.3 恢复 pending 结果

主结果写入失败时可能产生 `*.pending.jsonl`，可幂等恢复：

```bash
python grok_register_ttk.py retry-pending <pending文件> [输出文件]
```

示例：

```bash
python grok_register_ttk.py retry-pending accounts_20260715_120000.txt.pending.jsonl
```

## 5. 常见问题

### 5.1 PowerShell 报错：`&&` 不是有效语句分隔符

**原因**：旧版 PowerShell 不支持 `&&`，且 `source` 是 Bash 语法。

**处理**：按 [3.1](#31-windows-powershell推荐本机日常使用) 分两行执行，或使用：

```powershell
.\.venv\Scripts\python.exe grok_register_ttk.py
```

### 5.2 提示无法加载 `Activate.ps1`

**原因**：PowerShell 执行策略限制。

**处理**：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

或跳过激活，直接用 `.\.venv\Scripts\python.exe`。

### 5.3 GUI 不可用 / 缺少 Tkinter

日志会提示改用 CLI：

```bash
python grok_register_ttk.py cli
```

### 5.4 配置校验失败

- GUI 能打开但点「开始注册」失败：多为 **运行校验** 未通过（例如选了 `yyds` 但未填 `yyds_api_key` / `yyds_jwt`）。
- 对照 `config.example.json` 与 README 中各邮箱、CPA、grok2api 配置说明补全字段。

### 5.5 未打开浏览器 / 浏览器启动失败

- 确认已安装 Chrome 或 Chromium。  
- 检查代理 `proxy` 是否可达。  
- 关闭占用异常的残留 Chromium 进程后重试。
